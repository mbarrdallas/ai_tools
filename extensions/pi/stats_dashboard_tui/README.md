# Stats Dashboard TUI

Real-time agent monitoring dashboard for Pi with metrics, tool history, and conversation tracking.

## Overview

The Stats Dashboard TUI is a Pi extension that provides a terminal-based overlay dashboard for monitoring running agents. It displays live metrics (tokens, cost, context usage), tool execution history with timing, and conversation flow—all in a tabbed interface that lets you track multiple agents and their subagents simultaneously.

## Features

- **📊 Real-time Metrics** - Token usage (input/output/cache), cost tracking, context window monitoring, turn count
- **🔧 Tool History** - Complete tool call log with execution status, timing, and error details
- **💬 Conversation View** - Message history with user/assistant exchanges
- **📑 Multi-agent Tabs** - Dedicated tab for each agent with visual hierarchy for subagents
- **🔔 Smart Notifications** - Alerts for high context usage (≥80%) and tool failures
- **⌨️ Keyboard-driven** - Full keyboard navigation and shortcuts for efficiency
- **🎯 Tab Dismissal** - Clean up completed/failed agents with a single keystroke

## Installation

The extension is part of the `ai_tools` repository and follows Pi's standard extension loading conventions.

### Automatic Loading

If installed in `~/WORKSPACE/REPOS/ai_tools/extensions/pi/stats_dashboard_tui/`, Pi will auto-discover and load it on session start if:

1. Your Pi config references the extensions directory
2. The `package.json` has `"pi": { "extension": true }`

### Manual Registration

If Pi doesn't auto-load, you can manually register the extension path in your Pi config file.

## Quick Start

### Opening the Dashboard

Use either:
- **Command:** `/stats` in the Pi chat
- **Keyboard shortcut:** `Ctrl+Shift+S`

The dashboard appears as an overlay on the right side of your terminal.

### Basic Usage

```bash
# Start Pi session
pi

# Open dashboard
/stats

# Or use keyboard shortcut
# Press Ctrl+Shift+S

# Navigate between agent tabs
# Press Tab or → to move right
# Press Shift+Tab or ← to move left

# Close dashboard
# Press ESC or q
```

```typescript
// When you spawn an agent, it appears in the dashboard automatically
pi> /spawn requirements-agent "gather requirements for feature X"

// Switch to the new agent's tab with Tab key
// View its metrics, tool calls, and conversation in real-time

// The dashboard tracks all agents until you dismiss them
```

### Tab Management

**Tab States:**
- 🟢 **Running** - Agent is actively working
- ✓ **Completed** - Agent finished successfully
- ✗ **Failed** - Agent encountered an error

**Dismissing Tabs:**
- Select a completed or failed agent tab
- Press `d` to remove it from the dashboard
- Running agents cannot be dismissed (safety feature)
- Dismissing a parent agent dismisses all its subagents

## Metrics Displayed

### Token Metrics

| Metric | Description |
|--------|-------------|
| **Input Tokens** | Tokens sent to the model (prompt + context) |
| **Output Tokens** | Tokens generated by the model (responses) |
| **Cache Read** | Tokens retrieved from prompt cache |
| **Cache Write** | Tokens written to prompt cache |

### Cost Tracking

Displays cumulative cost in USD based on model pricing and token usage. Updated after each LLM turn.

### Context Window

Visual progress bar showing:
- Current context usage (tokens)
- Model's context limit
- Percentage (with warning color at ≥80%)

Example: `████████████░░░░░ 75% (96k / 128k)`

### Turn Count

Number of complete request/response cycles with the LLM.

## Tool History View

The tool history section shows all tool executions in chronological order with:

- **Status icons:**
  - ⏳ **Pending** - Tool is currently executing
  - ✓ **Success** - Tool completed successfully
  - ✗ **Failed** - Tool encountered an error

- **Tool details:**
  - Tool name (e.g., `read`, `bash`, `write`)
  - Execution duration (e.g., `250ms`, `1.5s`)
  - Argument summary (automatically formatted per tool type)
  - Error message (for failed tools)

**Example entries:**
```
✓ read (250ms) - path: "src/index.ts"
✓ bash (1.2s) - command: "npm test"
✗ write (50ms) - path: "locked.txt" [Error: EACCES: permission denied]
⏳ grep - pattern: "TODO", path: "src/"
```

## Conversation View

Displays message history between user and assistant:

- **User messages** - Prompts and commands sent to the agent
- **Assistant messages** - Agent's text responses
- **Timestamps** - When each message was sent
- **Preview mode** - Shows first ~100 characters per message
- **Expandable** - Press `c` to toggle full message content

Empty state shows: `"No messages yet"`

## Notifications

The dashboard provides non-blocking notifications for important events:

### High Context Usage

Triggers when an agent's context window reaches ≥80% capacity.

```
⚠️ Main Agent: Context usage at 85% (109k / 128k tokens)
```

**Why it matters:** Helps prevent context overflow and unexpected truncation.

### Tool Call Failures

Triggers when any tool execution fails with an error.

```
❌ Tool 'write' failed: EACCES: permission denied
```

**Why it matters:** Alerts you to issues that might block agent progress.

**Dismissing notifications:**
- Notifications are non-blocking (don't interrupt work)
- Auto-dismiss after 10 seconds
- Click or press any key to dismiss immediately
- Won't re-notify for the same issue (deduplication)

## Architecture Overview

### Code Organization

The extension follows a clean separation between shared library code and Pi-specific integration:

```
ai_tools/
├── lib/shared/stats_dashboard_tui/     # Shared library (platform-agnostic)
│   ├── types.ts                        # TypeScript type definitions
│   ├── state/
│   │   ├── store.ts                    # In-memory data store
│   │   ├── state-manager.ts            # High-level state API
│   │   └── notification-manager.ts     # Notification logic
│   ├── ui/
│   │   ├── dashboard.ts                # Main dashboard component
│   │   ├── tabs.ts                     # Tab bar component
│   │   ├── metrics-display.ts          # Metrics panel
│   │   ├── tool-history.ts             # Tool call list
│   │   ├── conversation-view.ts        # Message history
│   │   ├── agent-panel.ts              # Agent detail panel assembly
│   │   └── controller.ts               # Dashboard visibility controller
│   ├── handlers/
│   │   └── events.ts                   # Pi event handlers
│   └── utils/
│       └── format.ts                   # Formatting utilities
│
└── extensions/pi/stats_dashboard_tui/  # Pi extension wrapper
    ├── index.ts                        # Extension entry point
    ├── package.json                    # Extension metadata
    └── README.md                       # This file
```

### Why This Separation?

**Shared library (`lib/shared/`):**
- Core business logic (state management, event processing)
- UI components (platform-agnostic TUI components)
- Utilities (formatting, validation)
- **Benefit:** Reusable across different platforms or environments

**Extension wrapper (`extensions/pi/`):**
- Pi-specific integration (lifecycle hooks, API calls)
- Command registration (`/stats`)
- Shortcut registration (`Ctrl+Shift+S`)
- **Benefit:** Thin adapter layer, easy to test and maintain

### Data Flow

```
Pi Events → Event Handlers → State Manager → Data Store
                                    ↓
                              UI Components ← Dashboard Controller
                                    ↓
                              TUI Rendering → Terminal Display
```

1. **Events:** Pi emits lifecycle events (agent start/end, message, tool execution)
2. **Handlers:** Event handlers extract metrics and update state
3. **State:** StateManager maintains agent data, metrics, tool history
4. **UI:** Components render current state to terminal
5. **Controller:** Manages dashboard visibility and refresh triggers

### Key Components

| Component | Responsibility |
|-----------|---------------|
| **StateManager** | High-level API for agent tracking, metrics, tool calls |
| **DataStore** | Low-level in-memory storage with efficient indexing |
| **NotificationManager** | Threshold checks, deduplication, notification display |
| **DashboardController** | Visibility state, render triggers, command/shortcut handling |
| **Event Handlers** | Extract data from Pi events, update state, detect subagents |
| **UI Components** | Render state to TUI elements, handle keyboard input |

### State Management

**Agent Lifecycle:**
1. `agent:start` event → Create agent in store
2. `message:end` event → Update metrics (tokens, cost, context)
3. `tool:execution:start` → Create tool call record
4. `tool:execution:end` → Complete tool call, calculate duration
5. `agent:end` event → Mark agent as completed/failed
6. User dismisses tab → Remove agent from store

**Subagent Detection:**
- Monitors `tool:execution:start` events for `subagent` tool
- Extracts subagent name from tool arguments
- Creates parent-child relationship in state
- Updates parent's `subagentIds` array

**Memory Management:**
- Agents persist until dismissed (no auto-cleanup)
- Dismissing parent agent cascades to children
- Completed agents remain visible for review
- Clean shutdown on session end

### Adding New Metrics

To add a new metric to the dashboard:

1. **Update types** (`lib/shared/stats_dashboard_tui/types.ts`):
   ```typescript
   export interface AgentMetrics {
     // ... existing fields
     newMetric: number;
   }
   ```

2. **Update event handler** (`lib/shared/stats_dashboard_tui/handlers/events.ts`):
   ```typescript
   async function handleMessageEnd(event: any) {
     const newMetric = event.usage?.newMetric ?? 0;
     stateManager.updateMetrics(agentId, { newMetric });
   }
   ```

3. **Update UI component** (`lib/shared/stats_dashboard_tui/ui/metrics-display.ts`):
   ```typescript
   render() {
     const { newMetric } = this.agent.metrics;
     return box(`New Metric: ${formatNumber(newMetric)}`);
   }
   ```

### Adding New Notifications

To add a new notification type:

1. **Update types** (`lib/shared/stats_dashboard_tui/types.ts`):
   ```typescript
   export type NotificationType = 
     | "context_high" 
     | "tool_failed"
     | "new_notification_type";
   ```

2. **Add notification logic** (`lib/shared/stats_dashboard_tui/state/notification-manager.ts`):
   ```typescript
   public checkNewCondition(agentId: string): void {
     if (/* condition */) {
       this.notify('new_notification_type', agentId, 'Message');
     }
   }
   ```

3. **Call from event handler** (`lib/shared/stats_dashboard_tui/handlers/events.ts`):
   ```typescript
   notificationManager.checkNewCondition(agentId);
   ```

## Troubleshooting

### Dashboard Not Appearing

**Symptom:** `/stats` command or `Ctrl+Shift+S` doesn't show dashboard

**Solutions:**
1. Verify extension loaded: Check Pi startup logs for errors
2. Check extension path: Ensure `extensions/pi/stats_dashboard_tui/` exists
3. Verify package.json: Should have `"pi": { "extension": true }`
4. Restart Pi: Try `exit` then restart session

### Metrics Not Updating

**Symptom:** Dashboard shows but metrics are stuck at zero

**Solutions:**
1. Generate activity: Run a command or spawn an agent
2. Check event handlers: Verify handlers are registered
3. Inspect state: Use `console.log` in event handlers to debug
4. Try refresh: Press `r` in dashboard

### Tab Dismissal Not Working

**Symptom:** Pressing `d` doesn't dismiss tab

**Solutions:**
1. Check agent status: Only completed/failed agents can be dismissed
2. Wait for completion: Running agents block dismissal
3. Select correct tab: Ensure the tab is selected (highlighted)
4. Check for subagents: Parent agents with running subagents may block

### High Memory Usage

**Symptom:** Pi session slows down over time

**Solutions:**
1. Dismiss old agents: Press `d` on completed tabs to free memory
2. Close dashboard: Press ESC when not actively monitoring
3. Restart session: Long-running sessions accumulate state
4. Limit subagents: Excessive spawning increases memory footprint

### Notifications Not Appearing

**Symptom:** Expected notifications don't show

**Solutions:**
1. Check thresholds: Context warning triggers at ≥80%
2. Verify tool errors: Ensure tools are actually failing
3. Check deduplication: Same notification won't repeat
4. Inspect logs: Look for notification manager errors

## Performance Considerations

### Rendering Performance

- **Target:** <100ms render time per update
- **Caching:** All components cache renders when state unchanged
- **Throttling:** UI updates throttled to prevent excessive redraws
- **Efficiency:** O(1) agent lookups via Map-based storage

### Memory Footprint

- **Per agent:** ~1-5 KB depending on tool call history
- **Per tool call:** ~200-500 bytes
- **Notifications:** ~100 bytes each (auto-dismiss reduces accumulation)
- **Total:** Typically <1 MB for normal sessions (10-20 agents)

### Scalability Limits

| Scenario | Limit | Behavior |
|----------|-------|----------|
| Concurrent agents | ~50 | Tab bar scrolls, no performance degradation |
| Tool calls per agent | ~500 | List scrolls, consider pagination if needed |
| Conversation entries | ~200 | Truncated preview mode prevents bloat |
| Notifications | ~100 | Auto-dismiss keeps list bounded |
